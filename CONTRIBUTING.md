# Contributing

This skill improves when people contribute realistic tests, failure cases, and examples.

Good contributions include:

- plausible ideas, strategies, prompts, skills, messages, or data claims that need testing
- examples where the skill recommended too large a test, skipped evidence type, or missed an approval gate
- eval prompts with clear pass/fail expectations
- small wording improvements that make evidence-design behavior easier to trigger

Please keep contributions focused on the skill's behavior. The goal is better judgment infrastructure, not more documentation for its own sake.

Open an issue with:

1. the idea, claim, or artifact being tested
2. what evidence type should matter
3. what failure mode you want it to avoid
