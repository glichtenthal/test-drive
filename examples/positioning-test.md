# Example: Positioning Test

## User Input

I think "judgment infrastructure for human-led AI" is the right positioning for my agent skills catalog, but I do not want to overfit to language I personally like.

## Test Drive Output Shape

## What We Are Testing

Whether "judgment infrastructure for human-led AI" creates clearer resonance than more generic frames like "AI productivity skills" or "prompt workflows."

## Evidence Type

- Primary: Human Reaction
- Secondary: Artifact Performance
- Why this route fits: the uncertainty is whether people understand, remember, and repeat the frame.

## Smallest Credible Test

Publish or send two short variants to a small audience: one using "judgment infrastructure" and one using a more familiar productivity frame.

## Artifact Needed

Two short post or message variants.

## Signals

Support signal: readers repeat the language, ask about judgment, or describe a concrete use case.

Weaken signal: readers respond only to generic productivity, summarization, or automation language.

What would change my mind: if the simpler frame gets more concrete replies and the judgment frame gets only abstract praise.
