# Example: Skill Evaluation Test

## User Input

Test drive this skill before I publish it. It claims to help users turn messy personal and professional context into a structured brief, but I am worried it might just summarize things.

## Test Drive Output Shape

## What We Are Testing

Whether the skill produces a thinking surface that separates facts, claims, assumptions, tensions, and next use instead of compressing the input.

## Evidence Type

- Primary: Artifact Performance
- Secondary: Reasoning
- Why this route fits: the artifact itself needs to be evaluated under realistic use cases.

## Smallest Credible Test

Run three eval scenarios: messy product notes, a meeting dump with unresolved trade-offs, and personal career sensemaking.

## Artifact Needed

Eval prompts and pass/fail criteria.

## Signals

Support signal: outputs preserve uncertainty, separate evidence levels, and avoid premature advice.

Weaken signal: outputs become generic summaries, motivational advice, or confident recommendations.

What would change my mind: if the skill only works on one category of input or requires hidden context to perform well.
